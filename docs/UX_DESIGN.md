# Dashboard UX contract

## Visual thesis

A dark, quiet bio-instrument that belongs beside frontier-model tools: graphite surfaces, precise
typography, one acid-lime action accent, and Viventium's nine colors used only as live data signals.

## Content plan

1. **Now:** all nine Current/Nature lanes and the private Inner state at first glance.
2. **Nature:** four transparent starting profiles with consequences stated before apply.
3. **Trail:** chronological typed changes without prompt content.
4. **Settings:** reaction instruction plus pause, reset, and erase controls.
5. **About:** one tasteful bridge to the full Viventium platform and its creator.

The dashboard is an operating surface, not a landing page. Marketing copy is confined to About.

## Interaction thesis

- Rows enter in a short stagger so the nine-band instrument reads as one system.
- Current markers and Nature-to-Current tails move smoothly; changed values pulse once.
- Dialogs disclose consequences before profile, reset, or erase mutations.

All motion is removed under `prefers-reduced-motion`. Every action uses native controls and has a
visible focus state. The layout is single-column by tablet width and remains usable at 320 px.

## Information architecture preserved from full Viventium

The portable face lift keeps the underlying product grammar: nine ordered lanes, Now versus Nature,
motion tails, active range words, per-band tuning, Inner state, health, reaction trail, profile,
reaction instruction, reset, pause, and erase. It changes presentation, not affect semantics.
