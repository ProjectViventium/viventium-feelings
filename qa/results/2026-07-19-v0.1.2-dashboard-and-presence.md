## Failure modes closed by this acceptance pass

- A focused field no longer freezes state polling globally.
- A failed Now/Nature write rolls the lane back instead of presenting an uncommitted value as true.
- A failed range save retains the user's draft and supports retry.
- Rapid lane edits serialize instead of racing their own expected version.
- Pointer cancellation reverts rather than committing an accidental touch-scroll move.
- Keyboard editing commits exactly once.
- Coincident Now/Nature hit regions resolve by their visible marker rows, so an upper-half press on
  Now cannot silently edit Nature.
- A pointer gesture cancels and supersedes a pending keyboard debounce on the same thumb; its final
  store value and visible value cannot diverge.
- Focus and pending writes use separate lane ledgers; moving among controls inside a lane cannot
  leak focus accounting or suppress later remote movement.
- Polling preserves a focused profile action, full-render lane saves restore the equivalent focused
  control, and the per-second freshness clock is intentionally not a live region.
- Theme preference is stored in the plugin's private data directory rather than a random-port
  browser origin.
- Claude status presence is explicit, reversible, host-scoped, and ownership checked.
- Concurrent status-presence writers serialize; external edits, dangling symlinks, and conflicting
  status lines fail closed without replacing user state.
- Claude settings are synced to disk before the compare-and-rename step, matching the state store's
  durable temporary-file pattern.
- MCP metadata marks status-presence changes, reset, and erase as destructive, with contract tests
  protecting all three controls.
- The supported optional-presence uninstall sequence is Remove V, optionally erase Feelings data,
  then native plugin uninstall.

## Honest capability boundary

This plugin targets Claude Code and Codex plugin hosts. It does not alter ordinary consumer
Claude/ChatGPT web conversations. Codex currently supplies native plugin identity surfaces but no
arbitrary plugin-owned TUI status segment; Claude Code supports a user-configurable command status
line, so V presence there remains explicit opt-in. A third-party CLI plugin does not own the host
operating system's tray; the full Viventium macOS helper owns that native surface.

Remote commit/tag, GitHub Actions, immutable release archive, and fresh remote install evidence are
recorded only after publication; this local report does not pre-claim them.
