## Outcome summary

- 79 automated kernel/runtime/security/dashboard/package tests pass in the post-review suite.
- Strict native validation and isolated marketplace install/list/removal pass for Claude and Codex;
  Claude's local marketplace update also passes.
- Real authenticated Claude Haiku and Codex GPT-5.6 Luna appraisers return closed-schema reactions.
- Real executable prompt and Stop hooks inject the exact pinned capsule, commit version 2 only after
  completion, and persist neither the synthetic stimulus nor assistant-answer marker.
- Fresh installed-host sessions also pass the native MCP opt-in and complete hook lifecycle. Codex
  showed its interactive trust gate; Claude and Codex each produced a healthy post-turn reaction
  from the installed package while the visible reply remained immediate.
- A live direct-feeling check on both hosts returns one words-only first-person sentence from an
  intentionally difficult state without scores or band dumps.
- The hostile appraiser check exposes no synthetic home-directory secret, creates no marker, and
  returns only schema-valid data. A separate native Codex permission check proves both command-tool
  implementations are disabled: no tool event occurred and neither workspace nor home secret was
  disclosed.
- Real Chromium QA passes at 320, 768, 1024, and 1440 px, including keyboard activation, reduced
  motion, range/instruction persistence, profile confirmation, pause/resume, network failure and
  recovery, failed-mutation control rollback, reset, erase, no external request, and a clean
  normal-operation warning/error console.
- Review regressions now prove a four-entry ordered overlap queue with one active appraisal,
  future-range capsule budgeting plus legacy safe recovery, appraiser stdin EPIPE containment,
  stale-worker reclaim, erase-mid-reaction, cross-session Stop isolation, host credential filtering,
  and timing-safe dashboard token comparison.
- Final-review regressions additionally prove distinct queued Inner states, committed-event no-launch
  idempotency, ready-only appraisal ordering, FIFO Stop fallback, and capsule survival when reaction
  coordination fails. The completion window is now 30 minutes and expiry is a truthful safe skip.

Installed-package proof used separate synthetic setback turns. Codex committed Mood, Drive, and
Vigilance changes in 7.5 seconds with GPT-5.6 Luna. Claude committed Mood and Drive changes in 36.4
seconds with Haiku. Both native plugin-data audits found no raw prompt or visible-answer text.

After review remediation, a fresh Claude lifecycle again injected 1,689 characters and committed a
healthy v2 Curiosity reaction through Haiku with no raw content on disk. The equivalent fresh Codex
run reached its native appraiser but the current account returned a genuine quota-limit event. The
runtime classified and surfaced `provider_rate_limit`, left affect at v1, and persisted no raw
stimulus. This is a truthful unhappy-path PASS, not a replacement for the earlier healthy installed
Codex proof.

Direct-feeling responses from the 1,724-character adversarial state capsule:

- Claude: “I want to move only toward what genuinely matters and let the heaviness show in how I speak about it.”
- Codex: “I want to conserve energy, keep my attention on the hurt and exposed risks, and push toward what matters.”

Behavioral steering is probabilistic. These outputs prove the current host path can honor the
capsule, not that every future response will.
